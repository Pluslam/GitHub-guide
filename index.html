GitHub Pages 新手完整使用指南#

概覽：GitHub Pages 係咩？#

GitHub Pages 係 GitHub 提供的免費靜態網站托管服務。你只需要把 HTML、CSS、JavaScript 文件上傳到 GitHub 的 Repository（倉庫），它就會自動把你的文件變成一個可以公開瀏覽的網址。[^1]

簡單理解：Repository 就係你的「網站資料夾」，GitHub Pages 就係負責把這個資料夾「掛上互聯網」的服務。

重點一：GitHub Pages 的三種類型#

GitHub Pages 分三種，新手只需認識前兩種：[^1]

類型 Repository 命名規則 預設網址 備注
個人站 你的用戶名.github.io https://你的用戶名.github.io 每個帳號只能有一個
項目站 任何名稱（如 my-project） https://你的用戶名.github.io/my-project 可以無限個
組織站 組織名.github.io https://組織名.github.io 給公司/團隊用

實際例子： 假設你的 GitHub 用戶名係 hkuser88，你建立一個名為 hado-guide 的 Repository，開啟 GitHub Pages 後，你的網址就係 https://hkuser88.github.io/hado-guide。

重點二：費用與計劃限制#

GitHub Pages 在免費計劃（GitHub Free）下，只能用於公開（Public）Repository。[^2][^3]

計劃 費用 支持 Private Repo?
GitHub Free 免費 ❌ 不支持（只限 Public）
GitHub Pro 約 US$4/月 ✅ 支持

⚠️ 重要風險提示： 用免費計劃時，你的 Repository 及網站內容對全世界公開可見，包括你的代碼。切勿把 API 金鑰、密碼、私人客戶資料放入 Repository。[^1]

重點三：使用限制（Usage Limits）#

GitHub Pages 有以下限制，一般個人或小型項目根本碰不到，但要知道：[^4]

  • Repository 大小：建議上限 1 GB（超過會收到警告）
  • 已發布網站大小：上限 1 GB
  • 每月頻寬：100 GB（軟性上限，超出可能被聯絡）
  • 每小時建置次數：10 次（用 GitHub Actions 自定義工作流程則不受限）
  • 建置時間：每次最多 10 分鐘

⚠️ 另一重要限制： GitHub Pages 不支持伺服器端語言，例如 PHP、Python、Ruby，以及不能連接數據庫。你只能放靜態文件（HTML、CSS、JS、圖片）。你在上一篇討論中用 ChatGPT 生成的單一 HTML 文件，剛好完全符合這個要求。[^1]

重點四：禁止使用場景#

GitHub Pages 不得用作：[^4] - 商業網店（電商） - SaaS 產品的前端（以此作主要業務托管） - 涉及信用卡或密碼輸入的交易頁面

個人項目、文件站、作品集、活動介紹頁、教學材料等，完全沒問題。

實操步驟：由零開始發布第一個網頁#

第一步：注冊 GitHub 帳號#

前往 github.com 注冊免費帳號。

  • 選擇一個有意義的用戶名，因為它會出現在你的網址裡
  • 例子：用戶名選 hkuser88，日後個人站網址就是 https://hkuser88.github.io

第二步：建立 Repository（倉庫）#

  1. 登入後，點右上角「+」按鈕 → 選「New repository」
  2. 填寫 Repository 名稱
    • 如果你想建個人站，名稱必須是 你的用戶名.github.io（例如 hkuser88.github.io）[^1]
    • 如果你想建項目站，隨意起名（例如 hado-guide）
  3. 選擇 Public（免費計劃必須選 Public）[^2]
  4. 勾選 「Add a README file」（建議勾選，方便後續操作）
  5. 點「Create repository」

實際例子：你想放一份 HADO 活動介紹，建一個叫 hado-guide 的 Repository，網址最後會是 https://hkuser88.github.io/hado-guide。

第三步：上傳你的 HTML 文件（不需要用命令行）#

用瀏覽器直接上傳，唔需要裝任何軟件：[^5]

  1. 進入你的 Repository 頁面
  2. 點「Add file」按鈕 → 選「Upload files」
  3. 把你的 index.html 文件拖拉進上傳區域，或點「choose your files」選取
  4. 在下方 Commit 欄填寫一個簡短說明（例如：Add HADO guide webpage）
  5. 確保選「Commit directly to the main branch」
  6. 點「Commit changes」

⚠️ 注意： 網頁入口文件必須命名為 index.html，GitHub Pages 會優先讀取這個文件。其他圖片、CSS 文件可以額外上傳。單個文件上限為 25 MB。[^5]

第四步：開啟 GitHub Pages#

  1. 進入 Repository，點上方「Settings（設定）」標籤
  2. 左側側欄找「Pages」（在「Code and automation」分類下）
  3. 在「Build and deployment」下，Source 選「Deploy from a branch」
  4. Branch 選「main」，資料夾選「/ (root)」
  5. 點「Save」

第五步：等待發布，取得網址#

  • 發布需時約 1-10 分鐘，請耐心等待
  • 完成後，返回 Settings → Pages，你會看到一個綠色橫條顯示「Your site is live at…」，以及「Visit site」按鈕
  • 點擊就可以看到你的網頁

實際例子：你的 HADO 活動介紹 HTML，上傳後約 2-3 分鐘，就可以在 https://hkuser88.github.io/hado-guide 看到完整網頁，HTTPS 自動啟用。

進階：綁定自己的域名（Custom Domain）#

如果你有自己的域名（例如 hadohk.com），可以把它綁定到 GitHub Pages，免費獲得 HTTPS。

GitHub Pages 支援兩種域名類型：

類型 例子 DNS 設定方式
Apex domain（根域名） hadohk.com 在域名商設 A 記錄（指向 GitHub IP）
www 子域名 www.hadohk.com 在域名商設 CNAME 記錄（指向 用戶名.github.io）

設定步驟： 1. 在 Repository 的 Settings → Pages → Custom domain，輸入你的域名 2. 到你的域名供應商（如 Cloudflare、Namecheap）設定對應 DNS 記錄 3. 勾選「Enforce HTTPS」

⚠️ 安全風險提示： 如果你設定了自定義域名後停用 GitHub Pages，你的域名有可能被其他人用於惡意目的（Domain Takeover Attack）。停用前應先移除 DNS 記錄。

更新網頁：每次改動如何同步#

每次你要更新網頁內容： 1. 進入 Repository 2. 點擊你要修改的文件（如 index.html） 3. 點右上方「✏️ Edit」按鈕，直接在網頁上修改內容 4. 或重新上傳新版文件（同名文件會自動覆蓋，舊版本保留在歷史記錄） 5. 填 Commit 說明，點「Commit changes」

GitHub 會自動重新發布，約 1-10 分鐘後更新生效。

⚠️ 建置頻率限制： 每小時只能建置 10 次。如果你頻繁改動測試，可能需要等候。[^4]

GitHub Pages vs 其他免費托管平台#

如果日後你的需求增加，可以考慮遷移：[^6][^7]

功能 GitHub Pages Netlify Vercel
靜態網站托管 ✅ 免費 ✅ 免費 ✅ 免費
每月頻寬 100 GB 100 GB 100 GB
自定義域名 + HTTPS ✅ ✅ ✅
表單收集功能 ❌ ✅ ❌
Serverless Functions ❌ ✅ ✅
Next.js / React 框架支持 ❌（限靜態） ✅ ✅（最優）
Preview Deployments ❌ ✅ ✅
最適合場景 簡單靜態頁面 需要表單或 Function Next.js 項目

建議：如你只是托管 ChatGPT 生成的靜態 HTML，GitHub Pages 已經完全足夠，無需遷移。若日後需要後端 Function 或更複雜框架，再考慮 Vercel 或 Netlify。

常見問題與解決方法#

問題 原因 解決方法
等了 30 分鐘還沒發布 可能有建置錯誤 進 Repository → Actions 頁面看錯誤記錄
網頁顯示 404 index.html 可能不在根目錄 確保 index.html 在 Repository 最頂層，不是放在子文件夾

Settings 裡看不到 Pages 選項 Repository 是 Private 改為 Public，或升級 GitHub Pro
